🐞 Webtest

Settings & usage. Everything stays in your browser — no server, no account required.

Step 1

Connect with your API token

Paste your ClickUp Personal API token. Find it in ClickUp under Settings → Apps → API Token (it starts with pk_).

Step 2

Choose where notes go

Pick the List that holds (or will hold) your parent task.

Step 3

How should captures be created?

Some teams file each bug as a subtask under one parent; others create a task directly in the list. Pick what fits this project.

Each capture becomes a subtask of this task:

Keyboard shortcuts

The fastest way to capture a bug. Shortcuts work on any page.

🎯 Pick an element
Highlight and click any element on the page to report a bug on it. Also available from the 🎯 button in the popup.
Ctrl + Shift + E
⌘ + Shift + E on Mac

Shortcuts are suggestions and may differ if another extension already uses them. View or change them at chrome://extensions/shortcuts.

Other ways to capture

  • Select text on any page → click the floating 💾 icon to capture it as a note.
  • Right-click selected text → “Save selection to Webtest” (text only, no screenshot).
